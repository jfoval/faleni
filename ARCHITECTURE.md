# How the Faleni repo fits together

One screen for a new contributor (or a fresh session). Everything flows from one
file.

## Data flow

```
lexicon.csv  ── the single source of truth (word,type,gloss,decomp,domain,notes)
   │
   ├─►  python3 tools/faleni.py            ← validate (legal sounds, no collisions,
   │       (zero dependencies)               onset-family compliance) + helpers
   │       also: lint-examples, contrasts,   Runs in CI on every change (the gate).
   │       free, family, suggest, check
   │
   └─►  python3 site/build.py              ← reads lexicon.csv + the .md docs,
           (markdown lib optional;            converts markdown, writes site/dist/
            built-in fallback if absent)
                   │
                   └─►  site/dist/  ──►  GitHub Pages   (deploy-pages.yml on push)
                                         https://jfoval.github.io/faleni/
```

## The map

| Path | What it is |
|---|---|
| `lexicon.csv` | **Source of truth.** Add/edit words here; everything else derives from it. |
| `tools/faleni.py` | Validator + word generator. Zero deps. The CI governance gate. |
| `SPEC.md`, `GRAMMAR-EXTRAS.md`, `COINAGE.md`, `compounds.md` | Reference docs. |
| `lessons/lesson-2..4.md` | Lessons 2–4 (markdown, audio-first via the `::cards` shortcode). |
| `site/build.py` | Static-site generator (`build_*` functions + a small markdown converter). |
| `site/templates/base.html`, `site/assets/` | Page shell, CSS, and JS (search, audio, card reveal). |
| `.github/workflows/` | `validate-lexicon.yml` (gate) · `deploy-pages.yml` (build + publish). |

## Two non-obvious things that will trip you up

1. **Lesson 1 has no markdown file.** It's *interactive* (audio for the 16 sounds),
   so it's generated by `build_sounds()` in `site/build.py`, not from a
   `lessons/lesson-1.md` (that file is just a stub pointing here). Lessons 2–4
   *are* markdown.
2. **`build.py` does not validate.** A local `python3 site/build.py` will happily
   build over a broken lexicon. **Always run `python3 tools/faleni.py` first**
   (CI does this for you on PRs).

## The everyday loop

```sh
python3 tools/faleni.py check siso      # legal? taken? right family?  (before coining)
#   edit lexicon.csv / lessons
python3 tools/faleni.py                 # must say OK - 0 problems
python3 tools/faleni.py lint-examples   # lesson tokens all resolve
python3 site/build.py                   # rebuild site/dist
git add -A && git commit && git push    # CI validates, site redeploys
```
